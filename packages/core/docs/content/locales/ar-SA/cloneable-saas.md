# القوالب

هل ترغب في شحن أداة التحليلات الخاصة بك والمدعومة بالذكاء الاصطناعي؟ عميل البريد؟ منشئ النماذج؟ اختر قالبًا، وستحصل على SaaS فعال في دقائق - الوكيل وقاعدة البيانات والمصادقة وخط أنابيب النشر المتصل بالفعل.

تمنحك معظم "القوالب" سقالة فارغة وقائمة TODO طويلة. الوكيل الأصلي يقلب ذلك. كل منتج هو **منتج كامل من فئة SaaS** — يمكن تشغيله بالفعل في اليوم الأول، ويمكن شحنه بالفعل، وهو ملك لك تمامًا للتخصيص والعلامة التجارية والنشر. فكر فيها على أنها SaaS قابلة للاستنساخ، وليست مجموعات أولية: فأنت تقوم بتكوين منتج نهائي، ولا تحدق في النموذج المعياري.

## النماذج المتاحة {#catalog}

كل تطبيق هو تطبيق حقيقي يمكنك استخدامه اليوم، ومنصة الإطلاق لنسختك الخاصة منه.

| القالب                                    | ما هو                                                                                                                                      |
| ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| [**Chat**](/docs/template-chat)           | الحد الأدنى من تطبيق الدردشة الأول مع سلاسل المحادثات الدائمة، وactions، والمصادقة، ومسار نظيف إلى UI المخصص أو الواجهة الخلفية الخاصة بك. |
| [**Mail**](/docs/template-mail)           | عميل خارق للطبيعة. البريد الوارد، والتسميات، وفرز الذكاء الاصطناعي، ولوحة المفاتيح أولاً، والمسودات والإرسالات من خلال الوكيل.             |
| [**Calendar**](/docs/template-calendar)   | Google Calendar الوكيل الأصلي. الأحداث، والمزامنة، وروابط الحجز العامة، والجدولة التي يحركها الوكيل.                                       |
| [**Content**](/docs/template-content)     | سبج مفتوح المصدر لـ MDX. Markdown/MDX محلي، محرر Tiptap، مزامنة Notion، تعاون متعدد المستخدمين في الوقت الفعلي.                            |
| [**Brain**](/docs/template-brain)         | دردشة واضحة للشركة مدعومة بالذاكرة المؤسسية المستشهد بها، والمصادر المعتمدة، وبوابات المراجعة، والاستشهادات.                               |
| [**Assets**](/docs/template-assets)       | مدير الأصول الرقمية لمكتبات العلامات التجارية، والتحميلات، والمراجع، وإنشاء الصور/مقاطع الفيديو المتعلقة بالعلامة التجارية.                |
| [**Slides**](/docs/template-slides)       | عروض Google التقديمية الأصلية للوكيل. التشكيلات المستندة إلى React يقوم الوكيل بإنشائها وتحريرها مباشرة.                                   |
| [**Video**](/docs/template-videos)        | رسومات متحركة آلية ومقاطع فيديو توضيحية للمنتج على Remotion.                                                                               |
| [**Analytics**](/docs/template-analytics) | السعة/Mixpanel الأصلية للوكيل. قم بتوصيل مصادر البيانات، والمطالبة بالمخططات، والتثبيت على لوحات المعلومات.                                |
| [**Clips**](/docs/template-clips)         | شاشة غير متزامنة + تسجيل بالكاميرا مع النسخ والفصول وملخصات الذكاء الاصطناعي.                                                              |
| [**Design**](/docs/template-design)       | استوديو النماذج الأولية HTML للوكيل الأصلي لتصميمات Alpine/Tailwind التفاعلية.                                                             |
| [**Forms**](/docs/template-forms)         | نموذج أصلي للوكيل. يمكنك إنشاء عمليات الإرسال ومشاركتها وجمعها وتوجيهها إلى Slack أو جداول البيانات أو webhooks أو Discord.                |
| [**Plan**](/docs/template-plan)           | الخطط المرئية وملخصات العلاقات العامة مع المخططات والإطارات السلكية والتعليقات التوضيحية.                                                  |
| [**Dispatch**](/docs/template-dispatch)   | مستوى التحكم في مساحة العمل: الأسرار المشتركة، عمليات التكامل القابلة لإعادة الاستخدام، Slack/Telegram، المهام المجدولة.                   |

ألا تريد قالب المجال؟ استخدم [Chat](/docs/template-chat) عندما تريد تطبيقًا أساسيًا يمكن للمستخدمين التحدث إليه على الفور، أو بدء الإجراء أولاً باستخدام [Pure-Agent Apps](/docs/pure-agent-apps).

اطلع على الكتالوج الكامل ضمن [Templates](/templates)، أو انتقل مباشرة إلى أحد الكتالوجات - على سبيل المثال، [Dispatch](/docs/template-dispatch) هو مكان رائع للبدء إذا كنت تريد تطبيقًا على طراز مساحة العمل.

## ما الذي تحصل عليه خارج الصندوق {#what-you-get}

يأتي كل قالب مزودًا بالأجزاء التي يستغرق إنشاؤها عادةً شهورًا:

- **وكيل عامل** — تم توصيله بالفعل بالتطبيق، وهو قادر بالفعل على أخذ actions على بياناتك، ويدرك بالفعل سياق ما تبحث عنه. راجع [Messaging the agent](/docs/messaging) لمعرفة كيفية عمله.
- **المصادقة** — تسجيل الدخول، الجلسات، المؤسسات، عزل المستأجرين المتعددين. تم بالفعل.
- **قاعدة بيانات** — يحتوي كل قالب على مخططه واستعلاماته وعمليات ترحيله الجاهزة للاستخدام. أحضر قاعدة بيانات SQL الخاصة بك (Postgres، SQLite، Turso، D1) - يتكيف إطار العمل.
- **UI في الوقت الفعلي** — تظل الشاشة متزامنة مع ما يفعله الوكيل. انقر على "صياغة رسالة بريد إلكتروني" في الدردشة، وشاهد المسودة تظهر في بريدك الوارد على الفور.
- **جاهز للنشر** — ادفع إلى Netlify، أو Vercel، أو Cloudflare، أو AWS، أو أي مكان آخر يقوم بتشغيل Node. لا يوجد قفل للبائع.
- **عناصر ربط العلامة التجارية** — من السهل تغيير الاسم والألوان والشعار والنسخة.

هذا ليس ادعاءً نظريًا. يقوم مؤلف الإطار بتشغيل صندوق الوارد الفعلي الخاص به في قالب البريد، والتقويم الفعلي الخاص به في قالب التقويم، والتحليلات الفعلية الخاصة به في قالب التحليلات. القوالب هي برامج تشغيل يومية.

1. **اختر نموذجًا.** استخدم منتقي CLI، أو تصفح المستندات واختر واحدًا للبدء منه.
2. **العلامة التجارية.** قم بتغيير الاسم والألوان والشعار والنسخة. تعرض معظم النماذج ذلك في ملف تكوين واحد.
3. **قم بتخصيصه.** اطلب من الوكيل إضافة العمود الذي تحتاجه، وتغيير كيفية اتصال مجموعات البريد الوارد بـ API الداخلي، وإضافة طريقة عرض جديدة. يقوم الوكيل بتحرير الكود؛ قمت بمراجعة الفرق.
4. **قم بشحنه.** قم بتشغيل أمر النشر. لديك الآن SaaS للإنتاج الخاص بك في نطاقك الخاص.

عادةً ما تستغرق الخطوات من 2 إلى 4 أيامًا، وليس أشهرًا. الخطوة 3 مفتوحة - تتطور SaaS المتشعبة لديك بمرور الوقت، باللغة الإنجليزية البسيطة، من خلال التحدث إلى الوكيل.

## لماذا يعد هذا أمرًا عمليًا {#why}

ينهار نموذج fork-the-codebase التقليدي على نطاق واسع: كل مستخدم يحتفظ بصندوق الوارد الخاص به يبدو وكأنه كابوس صيانة. هناك قراران إطاريان يجعلان الأمر ناجحًا:

1. **يقوم الوكيل بالصيانة.** أنت لا تكتب تعليمات برمجية لإضافة عمود أو توصيل تكامل جديد — بل تطلب من الوكيل. لذا فإن "البريد الوارد المتشعب الخاص بك" يعد ميزة وليس عبئًا.
2. **التخصيص لكل مستخدم بدون رمز لكل مستخدم.** Skills والذاكرة والتعليمات وخوادم MCP المتصلة والوكلاء الفرعيين جميعهم يعيشون في SQL. يحصل كل مستخدم على طبقة التخصيص الخاصة به؛ تستضيف قاعدة التعليمات البرمجية المشتركة كل هذه العناصر مرة واحدة.

النتيجة: مرونة على مستوى الكود Claude لكل مستخدم، مع اقتصاديات نشر SaaS العادية.

## لا تريد الشوكة؟ {#hosted}

ليس عليك القيام بذلك. كل قالب متاح أيضًا كتطبيق مستضاف على `agent-native.com` — `mail.agent-native.com`، `calendar.agent-native.com`، وما إلى ذلك. استخدم الإصدار المستضاف مجانًا أو مدفوع؛ fork فقط عندما تريد تغيير شيء لا يكشفه الإصدار المستضاف.

## جرب ذلك بمهارة {#try-with-a-skill}

لست مستعدًا للسقالة؟ يمكنك إضافة قوى خارقة أصلية للوكيل إلى وكيل الترميز الذي تستخدمه بالفعل من خلال أمر واحد - دون الحاجة إلى تطبيق. شاهد [Skills Guide](/docs/skills-guide#app-backed-skills).

### للمطورين {#dev-details}

إذا كنت تعمل في مجال السقالات الآن، فإن أمر CLI هو:

```bash
npx @agent-native/core@latest create my-platform
```

سوف تحصل على منتقي متعدد التحديد. اختر تطبيقًا واحدًا (مستقلًا) أو عدة تطبيقات (مساحة العمل - تتشارك التطبيقات في المصادقة والعلامة التجارية وتكوين الوكيل وقاعدة البيانات). يتم دمج كل قالب تم اختياره في `apps/<name>/` مع كل ملف تحتاجه. بالنسبة إلى تطبيق الإجراء فقط بدلاً من القالب UI، استخدم `npx @agent-native/core@latest create my-agent --headless`.

املأ `.env` (غالبًا `ANTHROPIC_API_KEY` و`DATABASE_URL`)، و`pnpm install`، و`pnpm dev`، وسيعمل. لا يوجد "TODO: تنفيذ تسجيل الدخول،" ولا توجد مسارات نائبة.

أهداف النشر: أي مضيف متوافق مع Nitro (Node وCloudflare وNetlify وVercel وDeno وLambda وBun) وأي قاعدة بيانات SQL متوافقة مع SQL (SQLite وPostgres وTurso وD1 وSupabase وNeon). بالنسبة لمساحات العمل، تقوم `npx @agent-native/core@latest deploy` بإنشاء كل تطبيق في وقت واحد وشحنه خلف مصدر واحد. انظر [Deployment](/docs/deployment).

لتأليف ونشر القالب الخاص بك، راجع [Creating Templates](/docs/creating-templates).
